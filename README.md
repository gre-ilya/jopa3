# docxform — GUI-шаблонизатор документов .docx (Qt5)

`docxform` открывает `.docx` с плейсхолдерами `{{имя}}` и `\table{имя}`,
показывает форму с полем ввода на каждую переменную и выпадающим списком на
каждую таблицу, и сохраняет готовый `.docx` с подставленными значениями.

## Как это работает

Графическая программа на C++ / Qt5. Работает по принципу Jinja:

1. В Word/Google Docs/LibreOffice вы пишете документ и помечаете изменяемые
   места плейсхолдерами в двойных фигурных скобках, имя — латиницей, напр.:
   «Уважаемый `{{client_name}}`, ваш заказ `{{order_id}}` — `{{status}}`.»
2. `docxform` открывает `.docx`, находит все `{{имя}}` и показывает окно с
   полем ввода на каждую переменную (подпись поля = имя переменной).
3. Вы вводите значения и нажимаете «Создать документ…».
4. На выходе — новый `.docx`, где каждое `{{имя}}` заменено на введённое
   значение, а сами подставленные значения **выделены жёлтым**.

### Особенности

- **Группировка по абзацам.** Абзац документа показывается один раз (целиком —
  и текст до переменных, и после), а под ним идут поля ввода для всех его
  переменных. В тексте абзаца эти переменные подсвечены жёлтым, остальные
  `{{…}}` — серым. Так понятен контекст, а длинный абзац не повторяется для
  каждой переменной. Переменная относится к абзацу своего первого вхождения и
  в форме встречается один раз.
- **Подсказки.** У каждого поля есть tooltip с подробным описанием переменной.
- **Разорванные плейсхолдеры.** Плейсхолдеры, разбитые редактором на несколько
  фрагментов (частое явление в Word), собираются и заменяются корректно.
- **Сохранение форматирования.** Оформление подставляемого места (полужирность,
  размер и т.п.) берётся из исходного плейсхолдера; к нему добавляется жёлтое
  выделение.
- Картинки, таблицы, табуляции и прочее оформление сохраняются без потерь;
  плейсхолдеры внутри таблиц тоже обрабатываются.
- Имя плейсхолдера: буквы/цифры/подчёркивание (`[A-Za-z0-9_]`); вокруг имени
  внутри скобок допускаются пробелы: `{{ order_id }}`.

## Таблицы: `\table{имя}`

Кроме переменных, в шаблоне можно вставлять таблицы:

1. В нужном месте документа пишете `\table{имя}` (имя — латиницей). Лучше
   помещать заглушку в **отдельный абзац**: весь абзац с `\table{...}` будет
   заменён на таблицу.
2. При запуске `docxform` находит все `\table{имя}` и в окне рядом с каждой
   показывает **выпадающий список** доступных видов таблиц (подпись = имя).
3. Вы выбираете, какой вид подставить (или «— не вставлять —»).
4. На выходе на месте заглушки появляется готовая таблица с рамками и
   полужирной шапкой; содержимое берётся из выбранного вида.

Виды таблиц (сколько строк/столбцов, имена столбцов и содержимое) задаются в
исходном коде — в файле `tablekinds.cpp`. Содержимое может быть как
фиксированным, так и вычисляться в момент генерации (см. демо «График
платежей»). Подробная инструкция «как добавить новый вид» — в шапке
`tablekinds.cpp`.

В комплекте есть демонстрационные виды:

- Пустая таблица 3×3;
- Прайс-лист (пример);
- Сотрудники (пример);
- График платежей (строки генерируются в коде).

**Модульность:** всё, что касается таблиц (структуры данных, реестр видов и
рендеринг OOXML), вынесено в `tablekinds.h` / `tablekinds.cpp` и не зависит от
остального `docxform` — этот модуль можно переиспользовать в другом
Qt-приложении (вызвать `tableKinds()` для списка видов и `buildTableXml()` для
рендеринга).

## Требования

- Компилятор C++ с поддержкой C++17.
- Qt5 (модуль Widgets) и заголовки к нему.
  - Ubuntu/Debian: `sudo apt install qtbase5-dev`
  - Fedora: `sudo dnf install qt5-qtbase-devel`
- `pkg-config` — нужен для сборки через Makefile (читает Qt-флаги).
  - Ubuntu/Debian: `sudo apt install pkg-config`
  - Fedora: `sudo dnf install pkgconf-pkg-config`
- Библиотека zlib и заголовки к ней.
  - Ubuntu/Debian: `sudo apt install zlib1g-dev`
  - Fedora: `sudo dnf install zlib-devel`

Проверить Qt: `pkg-config --modversion Qt5Widgets`

## Сборка

```sh
make docxform
# или вручную:
g++ -O2 -std=c++17 -fPIC docxform.cpp tablekinds.cpp -o docxform \
    $(pkg-config --cflags --libs Qt5Widgets) -lz
```

## Запуск

```sh
./docxform                 # откроется диалог выбора .docx
./docxform шаблон.docx     # сразу открыть указанный файл
```

Без графики (для скриптов/проверки) есть «тихий» режим — он делает то же
самое, но без окна; значения переменных и выбор таблиц берутся из аргументов:

```sh
./docxform --render <вход.docx> <выход.docx> \
    ИМЯ=ЗНАЧЕНИЕ ... [+ИМЯ_ТАБЛИЦЫ=id_вида ...]
```

Аргумент вида `+имя=id_вида` подставляет в `\table{имя}` таблицу с данным id
(id видов см. в `tablekinds.cpp`: `blank3x3`, `price`, `employees`,
`schedule`). Пример:

```sh
./docxform --render in.docx out.docx "client=ООО Ромашка" \
    "+goods=price" "+plan=schedule"
```

Посмотреть найденные переменные с их контекстом и таблицы (то, что покажет
GUI):

```sh
./docxform --vars <вход.docx>
```

## Встраивание окна в другую программу

Окно `docxform` можно открыть прямо из своего Qt-приложения, не запуская
отдельный бинарник. Публичный интерфейс объявлен в `docxform.h`:

```cpp
namespace docxform {
    // Строит окно-форму для шаблона .docx и возвращает его как QWidget.
    // Владелец виджета — вызывающий код (показать через show(), при желании
    // задать родителя). При ошибке возвращает nullptr и кладёт текст ошибки
    // в *error. QApplication к моменту вызова уже должен существовать.
    QWidget* openTemplateForm(const QString& templatePath,
                              QString* error = nullptr,
                              QWidget* parent = nullptr);
}
```

Пример использования:

```cpp
#include "docxform.h"
...
QString err;
if (QWidget* form = docxform::openTemplateForm("contract.docx", &err)) {
    form->setAttribute(Qt::WA_DeleteOnClose);  // удалится при закрытии
    form->show();
} else {
    QMessageBox::critical(this, "Ошибка", err);
}
```

Окно само спросит значения переменных, покажет выпадающие списки для таблиц
`\table{…}` и по кнопке «Создать документ…» запишет готовый `.docx` (с
диалогом сохранения). Дальше вмешательство хост-программы не требуется.

Сборка вместе со своим приложением: компилируйте `docxform.cpp` с макросом
`DOCXFORM_NO_MAIN` (он убирает встроенный `main()`, чтобы не конфликтовать с
вашим) и слинкуйте с `tablekinds.cpp`:

```sh
g++ -O2 -std=c++17 -fPIC -DDOCXFORM_NO_MAIN \
    моя_программа.cpp docxform.cpp tablekinds.cpp \
    $(pkg-config --cflags --libs Qt5Widgets) -lz
```

Без `DOCXFORM_NO_MAIN` `docxform.cpp` собирается как самостоятельная программа
(встроенный `main` с GUI и режимами `--render` / `--vars`). Набор доступных
видов таблиц настраивается, как обычно, в `tablekinds.cpp`.

## Коды возврата (режим `--render`)

- `0` — успешно.
- `1` — ошибка (неверные аргументы, файл не читается / не `.docx`, или нельзя
  записать вывод).
