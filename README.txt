docx2txt / txt2docx / docxform — шаблонизатор документов
========================================================

Инструменты для шаблонов документов:

  docx2txt:  .docx  ->  .txt-шаблон   (вынимает выделенное жёлтым в переменные)
  txt2docx:  .txt-шаблон + значения  ->  готовый .docx
  docxform:  .docx с {{плейсхолдерами}} -> GUI-форма -> готовый .docx (Qt5)

Типичный сценарий:
  1. В Word/Google Docs пишете документ и выделяете ЖЁЛТЫМ те места, что
     меняются (имена, даты, суммы, номера).
  2. docx2txt превращает его в шаблон: выделенное -> VAR1, VAR2, ...,
     а в тексте остаётся \var{VAR1}.
  3. Заполняете значения переменных.
  4. txt2docx собирает готовый .docx с подставленными значениями.

=====================================================
ЧАСТЬ 1. docx2txt — извлечение текста абзацев из .docx в .txt
=====================================================

ОПИСАНИЕ
--------
CLI-программа на C++. Принимает .docx (сделанный в MS Word, Google Docs или
LibreOffice) и сохраняет в .txt текст абзацев документа, отдельно вынося
текст, выделенный жёлтым цветом, в переменные.

  - Содержимое таблиц НЕ извлекается.
  - Абзацы разделяются одной пустой строкой (как в .tex).
  - Длина каждой строки в секции TEXT не превышает 75 символов (перенос
    по словам).


ФОРМАТ ВЫХОДНОГО ФАЙЛА
---------------------
Файл состоит из четырёх секций:

  ========== IMAGES ==========
  IMAGE1=<стем>_images/image1.png
  IMAGE2=<стем>_images/image2.jpg

  ========== VARS ==========
  VAR1=текст, выделенный жёлтым
  VAR2=следующий выделенный фрагмент

  ========== SETTINGS ==========
  DEFAULT_FONT_SIZE=13
  HEADER_FONT_SIZE=16

  ========== TEXT ==========
  \ch{Заголовок главы}

  ... обычный текст абзаца, где каждый выделенный жёлтым фрагмент
      заменён на плейсхолдер \var{VAR1}, \var{VAR2}, ..., а каждая
      картинка — на \img{IMAGE1}, \img{IMAGE2}, ...

Картинки:
  - Каждая картинка из docx извлекается в папку <стем выходного txt>_images/
    (имя файла как в docx: image1.png и т.д.), путь пишется в секцию IMAGES.
  - В тексте на месте картинки ставится \img{IMAGEn}.
  - Картинки нумеруются по порядку появления в документе.

Главы и обычный текст:
  - Если абзац ЦЕЛИКОМ полужирный (в Word — Ctrl-B на всём абзаце), он
    считается главой и оборачивается в \ch{...}. Если полужирная только
    часть абзаца — это обычный текст, ничем не обрамляется.
  - Полужирность определяется по прямому форматированию run'ов: <w:b/> или
    <w:b w:val="1|true"/>.

Секция SETTINGS:
  - DEFAULT_FONT_SIZE — размер шрифта (в пунктах) обычного текста в .docx,
    который соберёт txt2docx.
  - HEADER_FONT_SIZE  — размер шрифта глав.
  - Это редактируемые значения по умолчанию (13 и 16); сам docx2txt их не
    вычисляет из входного файла.

Подробности по переменным:
  - Жёлтым считается стандартное выделение, как его сохраняют разные
    редакторы:
      <w:highlight w:val="yellow"/>  - выделение текста (MS Word);
      <w:shd w:fill="FFFF00"/>       - заливка (LibreOffice / Word);
      <w:color w:val="FFFF00"/>      - цвет шрифта (Google Docs).
  - Переменные нумеруются по порядку появления в документе: VAR1, VAR2, ...
  - Соседние выделенные фрагменты объединяются в одну переменную.
  - Выделение внутри таблиц игнорируется (таблицы не обрабатываются).
  - Если выделений нет, секция VARS остаётся пустой.
  - Перенос строк в секции TEXT считается по ИСХОДНОЙ длине текста, а
    плейсхолдер подставляется уже после переноса. Поэтому границы строк
    совпадают с исходным документом. Строки секции VARS не переносятся.
  - Значения в секции VARS пишутся «как есть», без обрамляющих скобок.


ШАБЛОННЫЙ .DOCX (<стем входного>_template.docx)
-----------------------------------------------
Помимо .txt docx2txt создаёт рядом с ним копию исходного .docx с суффиксом
_template (например, для contract.docx — contract_template.docx). Это тот же
документ, но каждый выделенный жёлтым фрагмент заменён прямо в тексте на имя
переменной в двойных фигурных скобках — {{VAR1}}, {{VAR2}}, ...

  - Скобки и имя переменной остаются выделенными тем же жёлтым, что и
    исходный фрагмент (форматирование run'а сохраняется).
  - Нумерация {{VARn}} совпадает с секцией VARS в .txt.
  - Соседние выделенные фрагменты дают одну переменную (как и в .txt).
  - Всё остальное (картинки, таблицы, форматирование) остаётся без изменений;
    выделения внутри таблиц не заменяются.
  - Файл собирается без сжатия (STORED), его открывают Word/LibreOffice.


ТРЕБОВАНИЯ
----------
  - Компилятор C++ с поддержкой C++17 (например, g++).
  - Библиотека zlib и заголовки к ней.
      Ubuntu/Debian:  sudo apt install g++ zlib1g-dev
      Fedora:         sudo dnf install gcc-c++ zlib-devel

  ВАЖНО про std::filesystem:
    На GCC 8 и старше (и старом Clang) std::filesystem вынесен в отдельную
    библиотеку, поэтому к команде сборки нужно добавить флаг -lstdc++fs
    (он должен идти ПОСЛЕ исходника, иначе линковщик его проигнорирует).
    Без него возникает ошибка линковки вида
    "undefined reference to std::filesystem::...".
    Начиная с GCC 9 этот флаг не нужен. Команды ниже включают его —
    на новых компиляторах он просто игнорируется.


СБОРКА
------
  Проще всего — через Makefile (он сам определит, нужен ли -lstdc++fs):
      make            # соберёт обе программы: docx2txt и txt2docx
      make docx2txt   # только docx2txt
      make clean      # удалить бинарники

  Либо вручную:
      g++ -O2 -std=c++17 -o docx2txt docx2txt.cpp -lz -lstdc++fs


ЗАПУСК
------
  ./docx2txt <входной.docx> <выходной.txt>

Пример:
  ./docx2txt input.docx output.txt

После выполнения текст абзацев будет записан в указанный .txt файл.


КОДЫ ВОЗВРАТА
-------------
  0  — успешно.
  1  — ошибка (неверные аргументы, файл не читается,
       файл не является корректным .docx, или нельзя записать вывод).


=====================================================
ЧАСТЬ 2. txt2docx — сборка готового .docx из шаблона и значений
=====================================================

ОПИСАНИЕ
--------
Берёт .txt-шаблон (вывод docx2txt) и подставляет значения вместо
плейсхолдеров \var{VAR1}, \var{VAR2}, ... На выходе — готовый .docx.


СБОРКА
------
  make            # соберёт обе программы сразу (см. часть 1)
  # или вручную:
  g++ -O2 -std=c++17 -o txt2docx txt2docx.cpp -lz -lstdc++fs


ЗАПУСК
------
  ./txt2docx <шаблон.txt> <значения.txt> <выход.docx>
  ./txt2docx <шаблон.txt> <выход.docx>        (только значения по умолчанию)


ФАЙЛ ЗНАЧЕНИЙ
------------
Текстовый файл, по одной переменной в строке:

  VAR1=Иван Петров
  VAR2=заказ №12345
  VAR3=ОПЛАЧЕНО

  - Можно скопировать секцию VARS из шаблона и отредактировать значения.
    Для совместимости со старыми шаблонами обрамляющие скобки {{ }}, если
    они есть, снимаются автоматически.
  - Пустые строки и строки, начинающиеся с '#', игнорируются.


ПОВЕДЕНИЕ ПОДСТАНОВКИ
--------------------
  - Если значение переменной не задано в файле значений, берётся значение
    по умолчанию из секции VARS шаблона. Поэтому запуск без файла значений
    (или с пустым файлом) воспроизводит исходный текст.
  - Если у переменной нет ни значения, ни дефолта — в stderr выводится
    предупреждение, а плейсхолдер \var{VARn} остаётся в документе как есть.
  - Перенос строк шаблона (75 символов) отменяется: строки одного абзаца
    снова склеиваются в сплошной текст, а перенос делает уже сам Word.

ГЛАВЫ И ШРИФТЫ В ВЫХОДНОМ .docx
------------------------------
  - Шрифт всего документа — Times New Roman.
  - Абзацы \ch{...} рендерятся как главы: ПОЛУЖИРНЫМ и размером
    HEADER_FONT_SIZE. Обрамление \ch{...} в итоговый текст не попадает.
  - Обычные абзацы — размером DEFAULT_FONT_SIZE.
  - Размеры берутся из секции SETTINGS шаблона; их можно переопределить,
    добавив DEFAULT_FONT_SIZE / HEADER_FONT_SIZE в файл значений.

ОТСТУП В НАЧАЛЕ АБЗАЦА
---------------------
  - Каждый ОБЫЧНЫЙ абзац начинается с одной табуляции (<w:tab/>), то есть с
    красной строки до первой позиции табуляции. Главы (\ch{...}) — без
    табуляции.

ИНТЕРВАЛЫ МЕЖДУ АБЗАЦАМИ
-----------------------
  - Между обычными абзацами интервал нулевой (before=0, after=0).
  - Каждая глава отделяется одним отступом строки сверху и снизу
    (before/after = одна строка тела, DEFAULT_FONT_SIZE). Word и
    LibreOffice складывают after предыдущего и before следующего абзаца,
    поэтому глава всегда обрамлена ровно одной пустой строкой с каждой
    стороны.

ЗАМЕЧАНИЕ О ФОРМАТИРОВАНИИ
-------------------------
Шаблон — это простой текст, поэтому из исходного оформления сохраняются
только структура глав (полужирность) и размеры шрифта. Остальное (таблицы,
цвета, выравнивание) НЕ переносится: txt2docx создаёт .docx с абзацами
текста, шрифтом Times New Roman и заданными размерами. Это сознательный
компромисс пары «docx -> txt -> docx».


КОДЫ ВОЗВРАТА
-------------
  0  — успешно.
  1  — ошибка (неверные аргументы, не читается шаблон/значения, в шаблоне
       нет секции TEXT, или нельзя записать .docx).


=====================================================
ЧАСТЬ 3. docxform — GUI-шаблонизатор в стиле Jinja (Qt5)
=====================================================

ОПИСАНИЕ
--------
Графическая программа на C++ / Qt5. Работает по принципу Jinja:

  1. В Word/Google Docs/LibreOffice вы пишете документ и помечаете изменяемые
     места плейсхолдерами в двойных фигурных скобках, имя — латиницей, напр.:
       «Уважаемый {{client_name}}, ваш заказ {{order_id}} — {{status}}.»
  2. docxform открывает .docx, находит все {{имя}} и показывает окно с полем
     ввода на каждую переменную (подпись поля = имя переменной).
  3. Вы вводите значения и нажимаете «Создать документ…».
  4. На выходе — новый .docx, где каждое {{имя}} заменено на введённое
     значение, а сами подставленные значения ВЫДЕЛЕНЫ ЖЁЛТЫМ.

Особенности:
  - Плейсхолдеры, разорванные редактором на несколько фрагментов (частое
    явление в Word), собираются и заменяются корректно.
  - Форматирование подставляемого места (полужирность, размер и т.п.) берётся
    из исходного оформления плейсхолдера; к нему добавляется жёлтое выделение.
  - Картинки, таблицы, табуляции и прочее оформление сохраняются без потерь;
    плейсхолдеры внутри таблиц тоже обрабатываются.
  - Имя плейсхолдера: буквы/цифры/подчёркивание ([A-Za-z0-9_]); вокруг имени
    внутри скобок допускаются пробелы: {{ order_id }}.


ТАБЛИЦЫ: \table{имя}
-------------------
Кроме переменных, в шаблоне можно вставлять таблицы. Принцип:

  1. В нужном месте документа пишете \table{имя} (имя — латиницей). Лучше
     помещать заглушку в ОТДЕЛЬНЫЙ абзац: весь абзац с \table{...} будет
     заменён на таблицу.
  2. При запуске docxform находит все \table{имя} и в окне рядом с каждой
     показывает ВЫПАДАЮЩИЙ СПИСОК доступных видов таблиц (подпись = имя).
  3. Вы выбираете, какой вид подставить (или «— не вставлять —»).
  4. На выходе на месте заглушки появляется готовая таблица с рамками и
     полужирной шапкой; содержимое берётся из выбранного вида.

Виды таблиц (сколько строк/столбцов, имена столбцов и содержимое) задаются
в ИСХОДНОМ КОДЕ — в файле tablekinds.cpp. Содержимое может быть как
фиксированным, так и вычисляться в момент генерации (см. демо «График
платежей»). Подробная инструкция «как добавить новый вид» — в шапке
tablekinds.cpp.

В комплекте есть демонстрационные виды:
  - Пустая таблица 3×3;
  - Прайс-лист (пример);
  - Сотрудники (пример);
  - График платежей (строки генерируются в коде).

Модульность: всё, что касается таблиц (структуры данных, реестр видов и
рендеринг OOXML), вынесено в tablekinds.h / tablekinds.cpp и не зависит от
остального docxform — этот модуль можно переиспользовать в другом Qt-приложении
(вызвать tableKinds() для списка видов и buildTableXml() для рендеринга).


ТРЕБОВАНИЯ
----------
  - Компилятор C++ с поддержкой C++17.
  - Qt5 (модуль Widgets) и заголовки к нему.
      Ubuntu/Debian:  sudo apt install qtbase5-dev
      Fedora:         sudo dnf install qt5-qtbase-devel
  - pkg-config — нужен для сборки через Makefile (читает Qt-флаги).
      Ubuntu/Debian:  sudo apt install pkg-config
      Fedora:         sudo dnf install pkgconf-pkg-config
  - Библиотека zlib (как у остальных инструментов).

  (pkg-config нужен только для docxform; docx2txt и txt2docx собираются и
   без него. Проверить Qt: pkg-config --modversion Qt5Widgets)


СБОРКА
------
  make docxform
  # или вручную:
  g++ -O2 -std=c++17 -fPIC docxform.cpp tablekinds.cpp -o docxform \
      $(pkg-config --cflags --libs Qt5Widgets) -lz


ЗАПУСК
------
  ./docxform                 # откроется диалог выбора .docx
  ./docxform шаблон.docx     # сразу открыть указанный файл

Без графики (для скриптов/проверки) есть «тихий» режим — он делает то же
самое, но без окна; значения переменных и выбор таблиц берутся из аргументов:

  ./docxform --render <вход.docx> <выход.docx> \
      ИМЯ=ЗНАЧЕНИЕ ... [+ИМЯ_ТАБЛИЦЫ=id_вида ...]

Аргумент вида +имя=id_вида подставляет в \table{имя} таблицу с данным id
(id видов см. в tablekinds.cpp: blank3x3, price, employees, schedule). Пример:

  ./docxform --render in.docx out.docx "client=ООО Ромашка" \
      "+goods=price" "+plan=schedule"


КОДЫ ВОЗВРАТА (режим --render)
------------------------------
  0  — успешно.
  1  — ошибка (неверные аргументы, файл не читается / не .docx, или нельзя
       записать вывод).
